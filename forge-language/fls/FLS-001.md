# FLS-001: Core Model

Status: Draft Baseline
Version: FLS-0
Category: Technical Specification
Normative Level: Level A

0. Applicability

This document defines the normative semantic model for Forge.

1. Computational Model
Forge computation SHALL proceed through deterministic transformations of valid Artifact Graphs.

2. Artifact
An Artifact consists of a persistent Identity and immutable Revisions.

3. Relationship
A typed directed association between two Artifacts.

4. Identity
Identity uniquely identifies one logical Artifact independent of revision or location.

5. Locator
Locators identify retrieval locations and SHALL NOT be used as Identity.

6. Lifecycle
Declared → Bound → Validated → Proven → Materialized → Deprecated → Archived.

7. Transformation
Match → Preconditions → Rewrite → Proof Obligations → Postconditions.

8. Proof
Proofs record claims, evidence, verifier, obligations, and status.

9. Manifest
Records provenance, specification version, transformations, proofs, and outputs.

10. Validity
A graph is valid if and only if schema validity, referential integrity, identity consistency,
lifecycle consistency, constraint satisfaction, and proof obligations are satisfied.
